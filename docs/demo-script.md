# 能力评估学生版演示脚本

## 1. 演示目标

本次演示展示的是一个学生可按步骤完成的职业能力自评原型：

```text
首页
-> 简历与理想岗位
-> 是否现在填写 48 题行为锚定自评问卷
-> 个人界面
```

一句话定位：

> 这个原型帮助学生先看到理想岗位需要哪些能力；如果完成自评问卷，系统会再把简历经历和问卷证据整理成个人能力雷达，用来对比当前优势、差距和下一步需要补充的行为证据。

## 2. 演示前准备

安装并启动前端：

```powershell
cd C:\code\career\agent\capability-assessment\app
npm install
npm run dev
```

启动 Ability API：

```powershell
cd C:\code\career
.\.venv\Scripts\python.exe agent\capability-assessment\rag-spike\scripts\ability_api_server.py
```

启动 RAG API：

```powershell
cd C:\code\career
.\.venv\Scripts\python.exe agent\capability-assessment\rag-spike\scripts\rag_api_server.py
```

打开 Vite dev server 显示的地址，通常是：

```text
http://localhost:5173
```

演示前确认：

- 本地已配置 `DEEPSEEK_API_KEY`。
- `.docx` 或文字版 `.pdf` 简历可解析；扫描 PDF 不支持。
- RAG 索引已构建。

## 3. 演示步骤与话术

### Step 1：首页

操作：

- 打开 Vite app。
- 点击“开始评估”。

话术：

> 这一版已经从静态三件套升级成独立 Vue/Vite 小应用，但仍然不接主系统登录和数据库。它不会做录用判断，也不是正式心理测评；报告只表示当前输入下的初步能力画像。

### Step 2：简历与理想岗位

操作：

- 上传 `.docx` 或文字版 `.pdf` 简历，或直接粘贴简历文本。
- 选择预置岗位：`互联网产品经理实习生` 或 `数据分析实习生`。
- 展示预置 JD，说明内容可见但不可修改。
- 切换到 `其他`，演示岗位名称和详细 JD 可以自行填写。

话术：

> 当前只保留简历、理想岗位和可选问卷。预置岗位可以降低学生输入成本；如果学生有自己的目标岗位，可以选“其他”并填写完整 JD，让岗位雷达更贴近真实目标。

强调：

- 上传文件由本地 Ability API 解析，浏览器不直接处理 Word/PDF。
- API 不保存上传文件。
- 扫描 PDF 会提示粘贴文字版简历。

### Step 3：问卷选择

操作：

- 点击“下一步”进入问卷选择页。
- 展示“现在填写问卷”和“稍后填写，进入个人界面”两个入口。
- 演示先选择“稍后填写”，进入个人界面。

话术：

> 问卷现在不是强制前置步骤。学生可以先看理想岗位能力要求，之后再回来补做能力测试。这样能减少第一次进入产品时的负担。

### Step 4：个人界面（未测试状态）

操作：

- 等待理想岗位雷达生成。
- 展示理想岗位能力雷达。
- 展示个人能力雷达空状态：“个人雷达还没有测试，暂时无法生成”。
- 点击“去填写问卷”。

话术：

> 个人界面会先呈现目标岗位的能力结构。因为学生还没有完成问卷，系统不会假装生成个人能力雷达，而是明确提示需要完成测试。

### Step 5：分页式问卷

操作：

- 每次回答 1 道题。
- 展示顶部进度 `当前题号 / 48`。
- 演示上一题和下一题修改。

话术：

> 问卷参考《Scale Development: Theory and Applications》的量表开发思路：先定义能力构念，再拆成行为指标，再写题项。每个能力维度有 3 个行为指标，每个指标 2 道题，共 48 题。题目尽量问行为频率、经历证据和可观察表现，不问“我觉得我很强”。

强调：

- 自评只能作为辅助证据。
- 每个维度最多保留 1 道反向题。
- 当前版本尚未完成正式信效度验证。

### Step 6：生成个人能力画像

操作：

- 完成 48 题后点击“生成报告”。
- 等待 AI 分析中页面完成。
- 回到个人界面。

话术：

> 如果前面已经生成过理想岗位雷达，这一步主要调用 Ability API，根据简历和问卷生成学生能力证据；如果岗位雷达还没有生成，也会同时调用 RAG API。现在不再静默生成 mock 报告；如果 API 或 key 没有准备好，页面会明确提示。

### Step 7：个人界面（已测试状态）

操作：

- 展示“个人能力雷达”和“理想岗位能力雷达”。
- 展示 Top 3 优势能力、Top 3 差距能力。
- 展示补充材料建议。
- 展开底部开发区，说明 JSON 对接结构。

话术：

> 报告不是说学生“能力好或差”，而是把能力倾向分和证据可信度分开。低可信度通常表示材料里缺少行为证据，下一步应该补项目背景、个人行动、结果数据或岗位关联，而不是给人格化评价。

## 4. 老师可能追问

问：问卷有什么专业依据？

答：

> 主参考书是《Scale Development: Theory and Applications》。当前按“构念定义 -> 行为指标 -> 题项池 -> 预测试 -> 题项分析 -> 信效度验证”的路线设计。这个原型已经完成构念、指标和题项结构化，但还没有完成正式信效度研究。

问：下一步如何验证问卷？

答：

> 下一步可以找 20-30 名学生预测试，记录每题理解难度、平均分、标准差、极端集中作答情况，再做内部一致性和题项筛选，并请指导老师审阅题项是否覆盖目标能力。

问：为什么不只看自评分？

答：

> 自评分容易受社会赞许和自我认知偏差影响，所以报告会把自评倾向和简历行为证据分开解释。高自评但缺少简历证据时，会提示补充行为证据。

## 5. 验收清单

- [ ] 前端通过 `npm run build`。
- [ ] 首页只有标题、用途、开始按钮和边界说明。
- [ ] 简历页包含上传/粘贴简历、预置岗位选择和 `其他` 自定义岗位。
- [ ] 预置岗位 JD 展示但不可编辑。
- [ ] `其他` 岗位名称和 JD 可编辑，缺少任一项时不能进入下一步。
- [ ] 上传 `.docx` 可提取文本。
- [ ] 上传文字版 `.pdf` 可提取文本。
- [ ] 扫描 PDF 显示限制提示。
- [ ] 用户可以选择立即填写问卷，也可以先进入个人界面。
- [ ] 未完成问卷时，个人界面展示理想岗位雷达和个人雷达空状态。
- [ ] 问卷每次只显示 1 题。
- [ ] 未完成 48 题不能生成个人能力雷达。
- [ ] 完成问卷后，个人界面展示个人能力雷达和理想岗位雷达。
- [ ] 个人界面展示优势、差距和补证据建议。
- [ ] JSON 默认隐藏在折叠开发区。
- [ ] Ability API 或 RAG API 不可用时不生成 mock 报告。

## 6. 当前限制

- 当前是独立 Vue/Vite 原型，不是生产系统。
- 不接主项目数据库、登录、学生档案或正式后端。
- 不做岗位推荐列表。
- 不做录用判断。
- 不做正式心理测评或医学诊断。
- 扫描 PDF 不做 OCR。
