# 真实样例采集与实测流程

本流程用于落实 6.15 会议提出的“用真实简历、真实 JD 和真实公司重跑主流程”。仓库不提交真实个人隐私数据；真实样例应在本地或受控环境中脱敏保存。

## 脱敏样例结构

每个样例建议使用以下结构记录：

```json
{
  "sample_id": "real_sample_001",
  "consent_status": "已获得测试授权 / 仅内部自测",
  "resume_text_redacted": "脱敏后的简历正文，不包含手机号、邮箱、学号、身份证号、家庭住址等隐私信息。",
  "target_company": "真实公司名，如需公开展示可脱敏为某电商平台/某教育科技公司",
  "target_role": "真实岗位名称",
  "target_jd": "真实 JD 正文或脱敏后的 JD 摘要",
  "role_source": "招聘网站 / 企业官网 / 老师提供 / 同学提供",
  "test_notes": "生成岗位模型、问卷、报告后的人工观察"
}
```

## 最小实测步骤

1. 使用脱敏真实简历，不使用“姓名 XX / 岗位 XX / 公司 XX”占位信息。
2. 使用真实 JD 或真实公司岗位描述，记录来源。
3. 在前端选择预置岗位或“其他”粘贴真实 JD。
4. 生成岗位能力模型，检查 6 个岗位维度是否贴合岗位。
5. 分别跑一次快速问卷、详细问卷或 AI 岗位问卷。
6. 检查报告中的关键差距、评价来源和 4 周提升计划是否可执行。
7. 记录明显不合理输出，并回填到下一轮 prompt、岗位指南或测试样例。

## 演示数据红线

- 不提交真实手机号、邮箱、证件号、住址、学号、精确生日。
- 不把未经授权的真实简历放入 git。
- 不使用明显虚构的占位姓名、占位岗位和占位公司作为主演示数据。
- 不把模型生成结果描述为 HR 或企业最终评价。
