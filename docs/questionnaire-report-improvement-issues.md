# 问卷与个人界面优化 Issues

> 来源：`docs/questionnaire-report-improvement-notes.md`。本文把会议反馈拆为可验收 issue，并记录实现状态。

## ISSUE-QA-01：修复问卷快速点击跳题

**Status**: Done

**What to build**

修复问卷选项快速点击时多次触发自动跳转的问题，保证一次题目作答最多只前进一题。

**Acceptance criteria**

- [x] 快速连续点击同一题选项，不会跨过中间题。
- [x] 快速切换同一题选项，只记录最后一次选择。
- [x] 自动前进只在当前题仍然是被点击题时发生。
- [x] 补答漏题后不需要连续点击多次下一题才能提交。

## ISSUE-QA-02：增加可收起答题导航栏

**Status**: Done

**What to build**

在问卷页增加可展开/收起的题号导航栏，展示已答、未答、当前题状态，并提供任意位置提交入口。

**Acceptance criteria**

- [x] 题号栏支持展开和收起。
- [x] 点击题号可以跳转到对应题。
- [x] 已答题变色并显示完成状态。
- [x] 未答题、当前题、已答题状态可区分。
- [x] 提交入口不依赖最后一题。
- [x] 未完成全部题目时不能提交，并定位第一个未答题。

## ISSUE-QA-03：增加 10 题快速作答模式

**Status**: Done

**What to build**

在问卷入口提供 10 题快速模式和 48 题详细模式。两种模式共用后续评估与个人界面流程。

**Acceptance criteria**

- [x] 问卷入口可选择快速模式或详细模式。
- [x] 快速模式只展示 10 题。
- [x] 详细模式展示 48 题。
- [x] 两种模式都能提交并进入个人界面。
- [x] 个人界面能显示本次问卷模式和题量。

## ISSUE-RP-05：个人界面问卷模式弹窗

**Status**: Done

**What to build**

个人界面进入问卷作答时，不再默认进入 48 题详细模式，改为弹窗选择快速模式或详细模式。

**Acceptance criteria**

- [x] 已生成个人雷达后重新作答会覆盖当前个人画像，不新增历史记录入口。
