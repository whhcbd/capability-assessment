# 问卷与个人界面优化 Issues

> 来源：`docs/questionnaire-report-improvement-notes.md`。本文把会议反馈拆为可验收 issue，并记录实现状态。

## ISSUE-QA-01：修复问卷快速点击跳题

**Status**: Done

**What to build**

修复问卷选项快速点击时多次触发自动跳转的问题，保证一次题目作答最多只前进一题。

**Acceptance criteria**

- [x] 快速连续点击同一题选项，不会跨过中间题。
- [x] 快速切换同一题选项，只记录最后一次选择。
- [x] 自动前进只在当前题仍然是被点击题时发生。
- [x] 补答漏题后不需要连续点击多次下一题才能提交。

## ISSUE-QA-02：增加可收起答题导航栏

**Status**: Done

**What to build**

在问卷页增加可展开/收起的题号导航栏，展示已答、未答、当前题状态，并提供任意位置提交入口。

**Acceptance criteria**

- [x] 题号栏支持展开和收起。
- [x] 点击题号可以跳转到对应题。
- [x] 已答题变色并显示完成状态。
- [x] 未答题、当前题、已答题状态可区分。
- [x] 提交入口不依赖最后一题。
- [x] 未完成全部题目时不能提交，并定位第一个未答题。

## ISSUE-QA-03：增加 10 题快速作答模式

**Status**: Done

**What to build**

在问卷入口提供 10 题快速模式和 48 题详细模式。两种模式共用后续评估与个人界面流程。

**Acceptance criteria**

- [x] 问卷入口可选择快速模式或详细模式。
- [x] 快速模式只展示 10 题。
- [x] 详细模式展示 48 题。
- [x] 两种模式都能提交并进入个人界面。
- [x] 个人界面能显示本次问卷模式和题量。

## ISSUE-RP-01：合并个人与职业雷达图

**Status**: Done

**What to build**

把个人能力雷达和职业要求雷达合并为同一张图，支持个人、职业、叠加三种显示模式。

**Acceptance criteria**

- [x] 默认显示个人与职业叠加视图。
- [x] 可切换只看个人、只看职业、两者叠加。
- [x] 两组图形样式清晰区分。
- [x] 切换不会改变评分数据。

## ISSUE-RP-02：雷达图显示具体分数

**Status**: Done

**What to build**

在雷达图中显示具体分数，减少用户在图和明细之间来回对照。

**Acceptance criteria**

- [x] 单图模式下显示每个能力维度分数。
- [x] 叠加模式下能区分个人分数和职业分数。
- [x] 分数标签不明显遮挡维度文本。

## ISSUE-RP-03：能力明细改为三大板块

**Status**: Done

**What to build**

将能力明细从表格式列表改为“岗位应用场景、个人能力评估、针对性改进建议”三块内容。

**Acceptance criteria**

- [x] 每个能力维度都有三块内容。
- [x] 岗位应用场景结合目标岗位要求。
- [x] 个人能力评估结合简历/问卷证据和分数。
- [x] 针对性改进建议具体到可执行动作。
